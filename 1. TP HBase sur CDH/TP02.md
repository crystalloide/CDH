## Les commandes HBase Shell

### Présentation

Dans cet TP, vous apprendrez les commandes HBase utiles pour gérer les fichiers. 

Vous explorerez les commandes générales, les commandes de gestion des tables, les commandes de manipulation des données, les outils de réplication de cluster et les outils de sécurité.

**Prérequis :** Vous devez avoir complété le TP01.

**Durée :** 30 minutes

---

### Tâches

**I. Dans la VM Cloudera Quickstart, dans un terminal de commande, exécutez les commandes suivantes pour se connecter en shell à l'instance HBase :**

```bash
export HADOOP_USER_NAME=hdfs
hbase shell
```

**II. Commandes générales HBase** — Explorons les différentes commandes disponibles dans HBase : 

1°) Commande `status` : dans l'instance HBase du conteneur Cloudera Quickstart, exécutez la commande `status` pour afficher l'état des serveurs HBase.

```hbase
status
```

2°) Commande `version` : affiche la version de HBase :

```hbase
version
```

3°) Commande `whoami` : affiche les informations sur l'utilisateur HBase courant :

```hbase
whoami
```

4°) Commande `table_help` : guide sur l'utilisation des commandes référencées par table et fournit la syntaxe des différentes commandes HBase Shell.

```hbase
table_help
```

**III. Commandes de gestion des tables**

1°) Commande `create` : utilisée pour créer une table HBase avec le nom spécifié selon le dictionnaire ou les spécifications de la famille de colonnes.

Syntaxe : `create <tablename>, <columnfamilyname>`

Dans le shell HBase, exécutez la commande suivante pour créer une table :

```hbase
create 'collaborateurs', 'Famille_de_colonne_1'
```

2°) Commande `list` : affiche toutes les tables présentes ou créées dans HBase.

```hbase
list
```
##### Affichage en retour : 
```texte
TABLE                       
collaborateurs   
1 row(s) in 0.0070 seconds

=> ["collaborateurs"]

```
##### Poursuivons :

3°) Commande `describe` : fournit des informations détaillées sur les familles de colonnes de la table indiquée.

Syntaxe : `describe <table name>`

```hbase
describe 'collaborateurs'
```

##### Affichage en retour : 
```texte

collaborateurs

```

4°) Commande `disable` : désactive la table

```hbase
disable 'collaborateurs'
```

5°) Commande `enable` : réactive la table désactivée

```hbase
enable 'collaborateurs'
```

6°) Commande `disable_all` : désactive toutes les tables correspondant à l'expression régulière donnée

```hbase
disable_all 'coll.*'
```

7°) Commande `is_enabled` : vérifie si la table nommée est activée ou non
```hbase
is_enabled 'collaborateurs'
```

8°) Commande `show_filters` : affiche tous les filtres présents dans HBase (ColumnPrefixFilter, TimestampsFilter, PageFilter, FamilyFilter, etc.)

```hbase
show_filters
```

9°) Commande `drop` : supprime la table présente dans HBase.

> **Note :** Assurez-vous de d'abord désactiver la table avec la commande `disable`.

```hbase
drop 'collaborateurs'
```

10°) Commande `drop_all` : supprime toutes les tables correspondant à l'expression régulière.

> **Note :** Assurez-vous d'avoir recréé la table avec `create 'collaborateurs', 'Famille_de_colonne_1'` après l'avoir supprimée à l'étape précédente. Vérifiez également que la table est bien désactivée avant d'utiliser `drop_all`.

```hbase
drop_all 'coll.*'
```

11°) Commande `alter` : utilisée pour ajouter/modifier/supprimer des familles de colonnes, ainsi que pour modifier la configuration de la table.

> **Note :** Assurez-vous d'avoir recréé la table avec `create 'collaborateurs', 'Famille_de_colonne_1'` après l'avoir supprimée à l'étape précédente.

12°) Ajouter/modifier une famille de colonnes :

```hbase
alter 'collaborateurs', NAME => 'f2', VERSIONS => 5
```

13°) Ajouter plusieurs familles de colonnes :

```hbase
alter 'collaborateurs', 'f2', {NAME => 'f3', IN_MEMORY => true}, {NAME => 'f4', VERSIONS => 5}
```

```texte
Chaque argument correspond à une opération distincte :

Argument	                        Opération	                      Effet
'f2'	                            Ajout de la famille f2	        Crée la famille de colonnes f2 avec les paramètres par défaut
{NAME => 'f3', IN_MEMORY => true}	Ajout de f3 avec cache mémoire	Les blocs de données de f3 sont gardés prioritairement dans le BlockCache
{NAME => 'f4', VERSION => 5}	    Ajout de f4 avec versioning	    HBase conserve jusqu'à 5 versions des cellules de f4 (au lieu de 1 par défaut)

Ce que ça implique concrètement
- IN_MEMORY => true sur f3 : indique à HBase de garder cette famille en cache RAM de manière prioritaire. C'est utile pour des données fréquemment lues (lookup tables, métadonnées).
- VERSION => 5 sur f4 : à chaque put, l'ancienne valeur n'est pas écrasée immédiatement — HBase conserve les 5 dernières valeurs horodatées, requêtables avec get 'collaborateurs', 'rowkey', {COLUMN => 'f4:col', VERSIONS => 5}.
- 'f2' en notation courte (sans accolades) est équivalent à {NAME => 'f2'}.

Remarque importante
La table collaborateurs doit être désactivée avant de lancer un alter structurel, ou à minima s'attendre à ce que HBase propage les changements région par région :
```

```text
disable 'collaborateurs'
alter 'collaborateurs', 'f2', {NAME => 'f3', IN_MEMORY => true}, {NAME => 'f4', VERSIONS => 5}
enable 'collaborateurs'
```


14°) Supprimer une famille de colonnes de la table :

```hbase
alter 'collaborateurs', 'delete' => 'f4'
```

15°) Définir la limite de taille de la table dans HBase :

```hbase
alter 'collaborateurs', MAX_FILESIZE=>'134217728'
```

```texte
Dans le contexte HBase/Hadoop (binaire) :

128 Mo = 134217728 octets (128 × 1 048 576 = 134 217 728)

En décimal SI : 128 × 1 000 000 = 128 000 000 octets,

mais la valeur binaire (134217728) est celle à utiliser dans les configurations HBase
(ex: hfile.block.cache.size, hbase.hregion.memstore.flush.size, etc.)

```


16°) Commande `alter_status` : affiche l'état de la commande `alter` — indique le nombre de régions de la table qui ont reçu le schéma mis à jour.

> **Note :** La mention `1/1 regions updated` signifie qu'une région a été mise à jour. En cas de succès, le message `done` s'affiche.

```hbase
alter_status 'collaborateurs'
```

**IV. Manipulation des données**

1°) Commande `put` : insère une valeur de cellule dans une table, une ligne et une colonne définies.

2°) Créer la table `collaborateurs1` :

```hbase
create 'collaborateurs1', 'PRENOM', 'NOM', 'IDENTIFIANT'
```

3°) Insérer des valeurs dans la table :

```hbase
put 'collaborateurs1','r1','PRENOM','Famille_de_colonne_1'
put 'collaborateurs1','r1','NOM','DURAND'
put 'collaborateurs1','r1','IDENTIFIANT','1890'
put 'collaborateurs1','r2','PRENOM','HERMIONNE'
put 'collaborateurs1','r2','NOM','DARTAGNAN'
put 'collaborateurs1','r2','IDENTIFIANT','1891'
```

4°) Commande `count` : récupère le nombre de lignes d'une table.

```hbase
count 'collaborateurs1'
```

5°) Commande `get` : récupère le contenu d'une ligne ou d'une cellule en passant le nom de la table, la ligne et optionnellement un dictionnaire de colonnes, d'horodatage, de plage temporelle et de versions.

```hbase
get 'collaborateurs1', 'r1', 'NOM'
```

6°) Commande `delete` : supprime la valeur de cellule dans la table/ligne/colonne spécifiée.

```hbase
delete 'collaborateurs1', 'r1', 'NOM'
```

7°) Commande `deleteall` : supprime toutes les cellules d'une ligne donnée.

```hbase
deleteall 'collaborateurs1', 'r1'
```

8°) Commande `truncate` : désactive, supprime et recrée la table spécifiée.

```hbase
truncate 'collaborateurs1'
```

9°) Commande `scan` : affiche le contenu de toute la table.

```hbase
scan 'collaborateurs1'
```

**V. Laissez le shell HBase ouvert pour le TP suivant.**
