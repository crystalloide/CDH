# TP04 : les APIs HBase (suite)

## Présentation

Nous allons configurer le serveur REST HBase, puis invoquer les APIs Admin et Client via l'outil Postman.

---

###

#Important : Au niveau de la VM (même en exécution), il faut ajouter une règle réseau NAT 8080:8080 pour ensuite pouvoir accéder via Postman

###

## Invocation de l'API HBase via Postman

### Présentation

Dans cette pratique, vous apprendrez à invoquer les Admin APIs et Client APIs REST Proxy HBase via Postman pour créer, supprimer et consulter les informations sur les namespaces et les tables dans HBase.

**Durée :** 30 minutes

**Prérequis :** Vous devez avoir complété le TP03

---

### Tâches

**1. Invoquer l'API Client REST Proxy HBase pour afficher le statut du cluster.**

a. Saisissez l'URL de requête ci-dessous avec la méthode GET et cliquez sur **Send** :

```text
http://localhost:8080/status/cluster
```

b. En cas de succès, vérifiez le statut du cluster affiché dans l'outil Postman.

---

**2. Invoquer l'API Client REST Proxy HBase pour afficher la version du cluster.**

a. Saisissez l'URL de requête ci-dessous avec la méthode GET et cliquez sur **Send** :

```text
http://localhost:8080/version/cluster
```

b. En cas de succès, vérifiez la version du cluster affichée dans l'outil Postman.

---

**3. Invoquer l'Admin API REST Proxy HBase pour créer un nouveau namespace nommé `samplenew`.**

a. Saisissez l'URL de requête ci-dessous avec la méthode POST et cliquez sur **Send** dans Postman :

```text
http://localhost:8080/namespaces/samplenew
```

b. En cas de succès, vérifiez la réponse de création affichée dans Postman.

---

**4. Invoquer l'API Client REST Proxy HBase pour afficher les namespaces disponibles dans HBase.**

a. Saisissez l'URL de requête ci-dessous avec la méthode GET et cliquez sur **Send** :

```text
http://localhost:8080/namespaces
```

b. En cas de succès, vérifiez la liste des namespaces affichée dans Postman.

---

**5. Invoquer l'Admin API REST Proxy HBase pour créer une nouvelle table nommée `users1`.**

a. Saisissez l'URL de requête ci-dessous avec la méthode POST :

```text
http://localhost:8080/users1/schema
```

b. Dans la section **Body**, sélectionnez le type de contenu **Raw** et le format **XML**.

c. Saisissez le contenu suivant dans le corps de la requête :

```xml
<?xml version="1.0" encoding="UTF-8"?>
<TableSchema name="users1">
  <ColumnSchema name="cf" />
</TableSchema>
```

d. Dans la section **Headers**, ajoutez le Content-Type suivant :

```xml
Key: Content-Type
Value: text/xml
```

e. En cas de succès, vérifiez la réponse de création affichée dans Postman.

> **Note :** Effacez le contenu de la section body après l'invocation.



**6. Invoquer l'API Client REST Proxy HBase pour obtenir des informations sur la table dans HBase.**

a. Saisissez l'URL de requête ci-dessous avec la méthode GET et cliquez sur **Send** :

```xml
http://localhost:8080/users1/schema
```

b. En cas de succès, vérifiez les informations sur la table affichées dans Postman.

---

**7. Invoquer l'API Client REST Proxy HBase pour obtenir les régions de la table disponibles dans HBase.**

a. Saisissez l'URL de requête ci-dessous avec la méthode GET et cliquez sur **Send** :

```xml
http://localhost:8080/users1/regions
```

b. En cas de succès, vérifiez les régions de la table affichées dans Postman.

---

**8. Invoquer l'Admin API REST Proxy HBase pour supprimer un namespace.**

a. Saisissez l'URL de requête ci-dessous avec la méthode DELETE et cliquez sur **Send** :

```xml
http://localhost:8080/namespaces/samplenew
```

b. En cas de succès, vérifiez que `OK` s'affiche dans Postman.

---

**9. Invoquer l'Admin API REST Proxy HBase pour supprimer une table.**

a. Saisissez l'URL de requête ci-dessous avec la méthode DELETE et cliquez sur **Send** :

```xml
http://localhost:8080/users1/schema
```

b. En cas de succès, vérifiez que `OK` s'affiche dans Postman.

---

**10. Arrêtez l'instance du serveur REST HBase et fermez l'outil Postman.**
