# TP05 : Utilisation de Sqoop, Pig et Hive avec HBase

## Présentation

Nous allons importer et exporter des données entre un SGBDR et HBase.

---

## Importation des données depuis un SGBDR vers HDFS vers HBase

### Présentation

Dans cette pratique, vous apprendrez à importer des données depuis un SGBDR vers HBase et depuis un SGBDR vers HDFS via Sqoop. Vous apprendrez également à importer des données depuis HDFS vers HBase.

**Prérequis :** Vous devez avoir complété le TP04

**Durée :** 30 minutes

---

## Scénario 1 : Importation des données depuis un SGBDR vers HBase

### Tâches

**I. Connexion à la base de données MySQL**

1°) Dans un terminal de commande de la VM **Cloudera Quickstart**, exécutez les commandes suivantes pour vous connecter à MySQL :

```bash
mysql -u root -p
```

Password: 

```bash
cloudera
```


2°) Dans le terminal MySQL, exécutez la commande suivante pour afficher toutes les bases de données disponibles :

```sql
show databases;
```

3°) Dans le terminal MySQL, exécutez la commande suivante pour utiliser la base de données `retail_db` :

```sql
use retail_db;
```

4°) Dans le terminal MySQL, listez toutes les tables présentes dans la base `retail_db` :

```sql
show tables;
```

5°) Dans le terminal MySQL, affichez toutes les colonnes de la table `departments` :

```sql
select * from departments;
```

6°) Dans le terminal Cloudera Quickstart, exécutez la commande suivante pour vous connecter au shell HBase :

```bash
hbase shell
```

7°) Dans le shell HBase, créez une table :

```hbase
create 'department', 'dept_details'
```

8°) Dans un nouveau terminal de commande la VM **Cloudera Quickstart**, exécutez la commande Sqoop Import suivante pour importer le contenu de la table `departments` depuis le SGBDR vers HBase :

```bash
sqoop import \
  --connect jdbc:mysql://quickstart.cloudera:3306/retail_db \
  --username root \
  --password cloudera \
  --table departments \
  --hbase-table department \
  --column-family dept_details \
  --hbase-row-key department_id -m 1
```

9°) En cas de succès, vérifiez que le message `Retrieved 6 records` s'affiche.

10°) Depuis le terminal Cloudera Quickstart, connectez-vous à HBase via `hbase shell`, puis exécutez la commande suivante pour afficher le contenu de la table :

```hbase
scan 'department'
```

## Scénario 2 : Importation des données depuis un SGBDR vers HDFS, puis de HDFS vers HBase

### Tâches

**I. Importer la table `products` depuis le schéma `retail_db` dans MySQL vers HDFS, puis de HDFS vers HBase.**

1°) **Import Sqoop depuis le SGBDR vers HDFS**

1.1°) Dans le terminal Cloudera Quickstart, exécutez la commande Sqoop suivante pour importer les enregistrements de la table `products` selon la condition donnée depuis le SGBDR vers HDFS :

```bash
sqoop import \
  --connect "jdbc:mysql://quickstart.cloudera:3306/retail_db" \
  --username=retail_dba \
  --password=cloudera \
  --query " select * from products where product_category_id=59 and \$CONDITIONS" \
  --target-dir=/user/cloudera/sqoop_import5 \
  --split-by products.product_id \
  --m 4 \
  --mapreduce-job-name "mysecondsqoopimport1" \
  --null-string "" \
  --null-non-string "" \
  --verbose
```

1.2°) En cas de succès, vérifiez que le message `24 records retrieved` s'affiche.

1.3°) Dans l'interface graphique HUE, en haut à gauche, allez sur le menu hamburger (3 barres horizontales) et naviguez vers **Browsers > Files** en haut à gauche pour ouvrir le navigateur de fichiers HDFS.

1.4°) Dans l'interface graphique HUE, naviguez vers le chemin `/user/cloudera/sqoop_import5` et vérifiez les 4 parties de fichiers mentionnées dans la commande Sqoop.

1.5°) Dans le navigateur de fichiers de HUE, ouvrez le fichier `part-m-00000` et consultez son contenu.

2°) Dans le shell HBase, exécutez la commande suivante pour créer une table :

```hbase
create 'products','cf1'
```

3°) Dans le terminal Cloudera Quickstart, exécutez la commande suivante pour importer la table depuis HDFS vers HBase :

```bash
hbase org.apache.hadoop.hbase.mapreduce.ImportTsv \
  -Dimporttsv.separator=',' \
  -Dimporttsv.columns='HBASE_ROW_KEY,cf1:product_id,cf1:product_category_id,cf1:product_name,cf1:product_description,cf1:product_price,cf1:product_image' \
  products /user/root/sqoop_import5/part-m-00000
```

4°) En cas de succès, vérifiez que tous les enregistrements d'entrée ont été transférés de HDFS vers HBase.

5°) Dans le shell HBase, exécutez la commande suivante pour afficher le contenu importé depuis HDFS :

```hbase
scan 'products'
```

## Scénario 3 : Importation des données depuis HBase vers Hive

### Tâches

**I. Importer la table depuis HBase vers Hive.**

1°) Dans le shell HBase, créez une table :

```hbase
create 'personal2', 'personal_data'
```

2°) Dans le shell HBase, insérez les données dans la table :

```hbase
put 'personal2', 1, 'personal_data:name', 'himagiri'
put 'personal2', 1, 'personal_data:age', '35'
put 'personal2', 2, 'personal_data:name', 'naveen'
put 'personal2', 2, 'personal_data:age', '31'
```

3°) Dans le terminal Cloudera Quickstart, connectez-vous au shell Hive :

```bash
hive
```

4°) Dans le shell Hive, exécutez la requête suivante pour créer une table externe Hive à partir de la table HBase :

```sql
CREATE EXTERNAL TABLE hbase_hive_personal13 (id INT, name STRING, age INT)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES ("hbase.columns.mapping" = ":key,personal_data:name,personal_data:age")
TBLPROPERTIES("hbase.table.name" = "personal2");
```

5°) En cas de succès, affichez le contenu de la table avec la requête select suivante :

```sql
select * from hbase_hive_personal13;
```

**Laissez le shell HBase et le terminal Cloudera Quickstart ouverts pour la pratique suivante.**
