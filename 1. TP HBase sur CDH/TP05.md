# TP05 : Utilisation de Sqoop, Pig et Hive avec HBase

## Présentation

Dans cette pratique, vous apprendrez à importer et exporter des données entre un SGBDR et HBase.

---

## Importation des données depuis un SGBDR vers HDFS vers HBase

### Présentation

Dans cette pratique, vous apprendrez à importer des données depuis un SGBDR vers HBase et depuis un SGBDR vers HDFS via Sqoop. Vous apprendrez également à importer des données depuis HDFS vers HBase.

**Prérequis :** Vous devez avoir complété les pratiques de la Leçon 3.

**Durée :** 90 minutes

---

## Scénario 1 : Importation des données depuis un SGBDR vers HBase

### Tâches

**1. Connexion à la base de données MySQL**

a. Dans le terminal Cloudera Quickstart, exécutez les commandes suivantes pour vous connecter à MySQL :

```bash
mysql -u root -p
```
Password: cloudera

text

b. Dans le terminal MySQL, exécutez la commande suivante pour afficher toutes les bases de données disponibles :

```sql
show databases;
```

c. Dans le terminal MySQL, exécutez la commande suivante pour utiliser la base de données `retail_db` :

```sql
use retail_db;
```

d. Dans le terminal MySQL, listez toutes les tables présentes dans la base `retail_db` :

```sql
show tables;
```

e. Dans le terminal MySQL, affichez toutes les colonnes de la table `departments` :

```sql
select * from departments;
```

f. Dans le terminal Cloudera Quickstart, exécutez la commande suivante pour vous connecter au shell HBase :

```bash
hbase shell
```

g. Dans le shell HBase, créez une table :

```hbase
create 'department', 'dept_details'
```

h. Dans le terminal Cloudera Quickstart, exécutez la commande Sqoop Import suivante pour importer le contenu de la table `departments` depuis le SGBDR vers HBase :

```bash
sqoop import \
  --connect jdbc:mysql://quickstart.cloudera:3306/retail_db \
  --username root \
  --password cloudera \
  --table departments \
  --hbase-table department \
  --column-family dept_details \
  --hbase-row-key department_id -m 1
```

i. En cas de succès, vérifiez que le message `Retrieved 6 records` s'affiche.

j. Depuis le terminal Cloudera Quickstart, connectez-vous à HBase via `hbase shell`, puis exécutez la commande suivante pour afficher le contenu de la table :

```hbase
scan 'department'
```

## Scénario 2 : Importation des données depuis un SGBDR vers HDFS, puis de HDFS vers HBase

### Tâches

**1. Importer la table `products` depuis le schéma `retail_db` dans MySQL vers HDFS, puis de HDFS vers HBase.**

a. **Import Sqoop depuis le SGBDR vers HDFS**

i. Dans le terminal Cloudera Quickstart, exécutez la commande Sqoop suivante pour importer les enregistrements de la table `products` selon la condition donnée depuis le SGBDR vers HDFS :

```bash
sqoop import \
  --connect "jdbc:mysql://quickstart.cloudera:3306/retail_db" \
  --username=retail_dba \
  --password=cloudera \
  --query " select * from products where product_category_id=59 and \$CONDITIONS" \
  --target-dir=/user/cloudera/sqoop_import5 \
  --split-by products.product_id \
  --m 4 \
  --mapreduce-job-name "mysecondsqoopimport1" \
  --null-string "" \
  --null-non-string "" \
  --verbose
```

ii. En cas de succès, vérifiez que le message `24 records retrieved` s'affiche.

iii. Dans l'interface HUE UI, naviguez vers **Manage HDFS** en haut à droite pour ouvrir le navigateur de fichiers.

iv. Dans le navigateur de fichiers HUE UI, naviguez vers le chemin `user/root/sqoop_import` et vérifiez les 4 parties de fichiers mentionnées dans la commande Sqoop.

v. Dans le navigateur de fichiers HUE UI, ouvrez le fichier `part-m-00000` et consultez son contenu.

b. Dans le shell HBase, exécutez la commande suivante pour créer une table :

```hbase
create 'products','cf1'
```

c. Dans le terminal Cloudera Quickstart, exécutez la commande suivante pour importer la table depuis HDFS vers HBase :

```bash
hbase org.apache.hadoop.hbase.mapreduce.ImportTsv \
  -Dimporttsv.separator=',' \
  -Dimporttsv.columns='HBASE_ROW_KEY,cf1:product_id,cf1:product_category_id,cf1:product_name,cf1:product_description,cf1:product_price,cf1:product_image' \
  products /user/root/sqoop_import5/part-m-00000
```

d. En cas de succès, vérifiez que tous les enregistrements d'entrée ont été transférés de HDFS vers HBase.

e. Dans le shell HBase, exécutez la commande suivante pour afficher le contenu importé depuis HDFS :

```hbase
scan 'products'
```

## Scénario 3 : Importation des données depuis HBase vers Hive

### Tâches

**1. Importer la table depuis HBase vers Hive.**

a. Dans le shell HBase, créez une table :

```hbase
create 'personal2', 'personal_data'
```

b. Dans le shell HBase, insérez les données dans la table :

```hbase
put 'personal2', 1, 'personal_data:name', 'himagiri'
put 'personal2', 1, 'personal_data:age', '35'
put 'personal2', 2, 'personal_data:name', 'naveen'
put 'personal2', 2, 'personal_data:age', '31'
```

c. Dans le terminal Cloudera Quickstart, connectez-vous au shell Hive :

```bash
hive
```

d. Dans le shell Hive, exécutez la requête suivante pour créer une table externe Hive à partir de la table HBase :

```sql
CREATE EXTERNAL TABLE hbase_hive_personal13 (id INT, name STRING, age INT)
STORED BY 'org.apache.hadoop.hive.hbase.HBaseStorageHandler'
WITH SERDEPROPERTIES ("hbase.columns.mapping" = ":key,personal_data:name,personal_data:age")
TBLPROPERTIES("hbase.table.name" = "personal2");
```

e. En cas de succès, affichez le contenu de la table avec la requête select suivante :

```sql
select * from hbase_hive_personal13;
```

**Laissez le shell HBase et le terminal Cloudera Quickstart ouverts pour la pratique suivante.**
